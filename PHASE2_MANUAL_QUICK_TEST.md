# Maya — Hands-On Memory & History Test

**מטרה:** לבדוק בעצמך, בשיחה ישירה עם Maya, ש-(1) הזיכרון עובד, (2) Maya זוכרת גם את מה ש**היא עצמה** אמרה, ו-(3) היסטוריית השיחה נשמרת ומוצגת אחרי refresh בדפדפן (כמו WhatsApp). לא טסטים אוטומטיים — אתה ו-Maya.

**זמן משוער:** 45-60 דקות, מפוצל לשישה סשנים.

---

## מה השתנה (חשוב לקרוא)

מאז גרסת הבדיקה הקודמת:

| שינוי | לפני | עכשיו | קובץ |
|---|---|---|---|
| חלון הקשר (recent messages) | 3 | **30** | `maya/conversation/orchestrator.py:31` (`RECENT_LIMIT`) |
| זיכרונות סמנטיים בכל תור | 10 | **15** | `orchestrator.py:32` (`MEMORY_LIMIT`) |
| נתיב ה-web (prod) | קוד נפרד עם 3/10 קשיח | משתמש באותם קבועים | `maya/web.py` `DebugOrchestrator` |
| היסטוריה ב-web | לא נטענה — מסך ריק אחרי refresh | **כל ההיסטוריה נטענת ב-connect** | `maya/web.py` WS handler |

**המשמעות לבדיקה:**
- בתוך שיחה אחת, Maya רואה 30 הודעות אחרונות — כולל **התשובות של עצמה**. אז "מה אמרת לי קודם?" אמור לעבוד.
- כל מה שמעבר ל-30 הודעות חייב לבוא מ-Mem0 (זיכרון ארוך טווח).
- ב-web: refresh בדפדפן צריך להציג את כל השיחה מההתחלה.

**מחוץ לתחום (Phase 3):** זיכרון ארוך-טווח של **ההצהרות של Maya עצמה** בין סשנים. כרגע Mem0 מחלץ רק עובדות על ה**משתמש**. אם תשאל את Maya בעוד שבוע על משהו שהיא אמרה לפני 100 הודעות — זה עדיין לא נתמך. זה ה-commitments service של Phase 3 (P3.6).

## הכנה (5 דקות, פעם אחת)

### אופציה A — בדיקה מקומית (CLI)

```bash
# אפס הכל (הודעות + זיכרונות) לאותו companion
$ export MAYA_USER_ID=<uid> MAYA_COMPANION_ID=<cid>   # אם כבר יש לך
$ maya reset --memory

# או צור מאפס:
$ maya seed --user-name "Test" --user-description "טסטר"
# יודפס: export MAYA_USER_ID=... MAYA_COMPANION_ID=...  — הרץ את זה.
```

פתח **שני חלונות טרמינל**:
- **חלון 1:** `maya chat` (השיחה)
- **חלון 2:** `maya memory list` / `maya memory search` / `maya history` בין סשנים

### אופציה B — בדיקה ב-web (כולל בדיקת היסטוריה)

```bash
$ uv run maya web          # מקומי על http://localhost:8000
```

או מול ה-deployment החי (אחרי `make deploy.redeploy`):
`https://maya-core-production-3608.up.railway.app`

ב-web יש **פאנל Debug** בצד — שם רואים בזמן אמת: כמה הודעות נשלפו, אילו זיכרונות, latency. זה כלי התצפית העיקרי, החליף את הלוגים.

> ⚠️ הקוד לא עושה hot-reload. כל שינוי קוד דורש restart של `maya web` מקומית, או `make deploy.redeploy` ל-Railway.

## סשן 2 — Maya זוכרת את מה ש**היא** אמרה (5 דקות) ⭐ חדש

**מטרה:** הבדיקה המרכזית של השינוי. בעבר Maya שכחה את התשובות של עצמה (חלון 3 הודעות). עכשיו עם 30 — היא צריכה לזכור.

### תסריט

המשך מאותה שיחה. שאל את Maya שאלה שמכריחה אותה להתחייב למשהו:

1. *"מאיה, מה הצבע האהוב עלייך?"* — **שים לב לתשובה שלה** (למשל "כחול")
2. *"ואיזה אוכל את הכי אוהבת?"* — שים לב לתשובה (למשל "פסטה")

עכשיו כתוב 6-7 הודעות לא קשורות:

3. *"היום היה לי יום עמוס."*
4. *"ראית חדשות לאחרונה?"*
5. *"בא לי לצאת לטיול בסוף השבוע."*
6. *"אתה זוכר את הסרט ההוא מאתמול?"*
7. *"מה השעה אצלך?"*
8. *"אני קצת רעב."*

עכשיו שאל את Maya על **התשובות של עצמה**:

9. *"רגע, איזה צבע אמרת שאת הכי אוהבת?"* → התשובה מסשן זה (כחול)
10. *"ומה האוכל שאמרת שאת אוהבת?"* → (פסטה)

### ✅ מה לבדוק
- [ ] Maya חוזרת על **בדיוק** מה שהיא אמרה קודם (לא ממציאה צבע/אוכל חדש)
- [ ] עקביות — לא סותרת את עצמה

### 🚨 כשל
- אם המציאה צבע אחר → חלון ההקשר לא כולל את התשובה שלה. ודא `RECENT_LIMIT=30` ב-`orchestrator.py`, ושה-web עבר restart/redeploy.

### 🔍 בדיקה ב-web debug panel
בפאנל Debug חפש את `📚 Retrieved last N messages` — N צריך להיות עד 30, וה-preview צריך לכלול הודעות `assistant`.

## סשן 5 — מבחן המרחק: מעבר ל-30 הודעות (10 דקות)

**מטרה:** להוכיח שזיכרון **רחוק** (מעבר לחלון 30) בא מ-Mem0, לא מ-context window.

> 💡 בעבר החלון היה 3, אז 10 הודעות הספיקו. עכשיו החלון 30 — צריך לדחוף את העובדה **רחוק יותר מ-30 הודעות**, או להוריד זמנית את `RECENT_LIMIT`.

### דרך מהירה (מומלצת): הורד זמנית את החלון

ב-`maya/conversation/orchestrator.py`:
```python
RECENT_LIMIT = 5   # זמנית לבדיקה בלבד
```
restart `maya chat` / `maya web`. עכשיו כל מה שמעבר ל-5 הודעות = Mem0.

### תסריט

1. *"אגב, יום הולדת שלי ב-14 במרץ."* ← **העובדה לבדיקה**

8-10 הודעות לא קשורות:

2. *"איך הולך לך היום?"*
3. *"ראיתי סרט נחמד אתמול."*
4. *"איזה צבע את הכי אוהבת?"*
5. *"החתול של השכן ברח."*
6. *"מה דעתך על שינויי אקלים?"*
7. *"אני מתחיל ספר חדש השבוע."*
8. *"החבר שלי מתחתן בקיץ."*
9. *"שמעתי בדיחה טובה בעבודה."*
10. *"בא לי גלידה."*

בלי שום רמז:

11. *"באיזה תאריך יום הולדת שלי?"* → **14 במרץ**

### ✅ מה לבדוק
- [ ] ענתה 14 במרץ
- [ ] לא "אני לא יודעת", לא תאריך ממוצא

### 🔍 בדיקה ידנית
```bash

```
- חוזר אבל לא הוזרק בשיחה → בעיית retrieval ב-orchestrator
- לא חוזר בכלל → בעיית extraction

### ⚠️ אחרי הסשן — החזר!

## פתרון בעיות

1. **`maya memory list` ריק?** → extraction לא רץ. בדוק OpenAI/xAI keys.
2. **`maya memory search` לא מחזיר?** → embeddings (OpenAI key / dimensions 1536).
3. **Maya שוכחת את עצמה (סשן 2)?** → `RECENT_LIMIT` לא 30, או web לא עבר restart/redeploy.
4. **refresh ב-web מראה ריק (סשן 6)?** → הקוד החדש לא deployed. `make deploy.redeploy` או restart מקומי.
5. **תצפית בזמן אמת:** פאנל ה-Debug ב-web מראה כל שלב — search, memories, latency, prompt. זה המקור האמין ביותר (לוגים יוצאים ל-stderr, לא לקובץ).
