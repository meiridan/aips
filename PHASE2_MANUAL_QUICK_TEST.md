# Maya — Hands-On Memory & History Test

**מטרה:** לבדוק בעצמך, בשיחה ישירה עם Maya, ש-(1) הזיכרון עובד, (2) Maya זוכרת גם את מה ש**היא עצמה** אמרה, (3) היסטוריית השיחה נשמרת אחרי refresh (כמו WhatsApp), ו-(4) **הביוגרפיה של Maya לא דולפת לזיכרון המשתמש** (הבאג שתוקן).

**זמן משוער:** 70-90 דקות, 10 סשנים ידניים + סוויטה אוטומטית.

**מבנה:** סשנים 1-6 = זיכרון בסיסי + היסטוריה. סשנים 7-10 = תרחישים מורכבים (אנטי-דליפה, בלבול זהויות, סתירות, מקרי קצה). בנוסף: בדיקה אוטומטית מהירה (`pytest`) שמכסה את הקריטיים.

---

## מה השתנה (חשוב לקרוא)

מאז גרסת הבדיקה הקודמת:

| שינוי | לפני | עכשיו | קובץ |
|---|---|---|---|
| חלון הקשר (recent messages) | 3 | **30** | `maya/conversation/orchestrator.py:31` (`RECENT_LIMIT`) |
| זיכרונות סמנטיים בכל תור | 10 | **15** | `orchestrator.py:32` (`MEMORY_LIMIT`) |
| נתיב ה-web (prod) | קוד נפרד עם 3/10 קשיח | משתמש באותם קבועים | `maya/web.py` `DebugOrchestrator` |
| היסטוריה ב-web | לא נטענה — מסך ריק אחרי refresh | **כל ההיסטוריה נטענת ב-connect** | `maya/web.py` WS handler |
| חילוץ זיכרון | חילץ גם מהודעת ה-assistant → **הביוגרפיה של Maya דלפה לזיכרון המשתמש** | מחלץ **רק מהודעת המשתמש** | `maya/memory/service.py` `extract_and_store` |

> 🐛 **באג שתוקן (חשוב):** Maya ממציאה ביוגרפיה בזמן roleplay ("אני בת 45, גרושה, 3 ילדים"). לפני התיקון, Mem0 חילץ את ההצהרות האלה ושמר אותן כעובדות על ה**משתמש** — שמות הילדים של Maya הופיעו כילדי המשתמש. עכשיו חילוץ הוא מהודעת המשתמש בלבד. **סשן 7-8 בודקים בדיוק את זה.**

**המשמעות לבדיקה:**
- בתוך שיחה אחת, Maya רואה 30 הודעות אחרונות — כולל **התשובות של עצמה**. אז "מה אמרת לי קודם?" אמור לעבוד.
- כל מה שמעבר ל-30 הודעות חייב לבוא מ-Mem0 (זיכרון ארוך טווח).
- ב-web: refresh בדפדפן צריך להציג את כל השיחה מההתחלה.

**מחוץ לתחום (Phase 3):** זיכרון ארוך-טווח של **ההצהרות של Maya עצמה** בין סשנים. כרגע Mem0 מחלץ רק עובדות על ה**משתמש**. אם תשאל את Maya בעוד שבוע על משהו שהיא אמרה לפני 100 הודעות — זה עדיין לא נתמך. זה ה-commitments service של Phase 3 (P3.6).

## הכנה (5 דקות, פעם אחת)

### אופציה A — בדיקה מקומית (CLI)

```bash
# אפס הכל (הודעות + זיכרונות) לאותו companion
$ export MAYA_USER_ID=<uid> MAYA_COMPANION_ID=<cid>   # אם כבר יש לך
$ maya reset --memory

# או צור מאפס:
$ maya seed --user-name "Test" --user-description "טסטר"
# יודפס: export MAYA_USER_ID=... MAYA_COMPANION_ID=...  — הרץ את זה.
```

פתח **שני חלונות טרמינל**:
- **חלון 1:** `maya chat` (השיחה)
- **חלון 2:** `maya memory list` / `maya memory search` / `maya history` בין סשנים

### אופציה B — בדיקה ב-web (כולל בדיקת היסטוריה)

```bash
$ uv run maya web          # מקומי על http://localhost:8000
```

או מול ה-deployment החי (אחרי `make deploy.redeploy`):
`https://maya-core-production-3608.up.railway.app`

ב-web יש **פאנל Debug** בצד — שם רואים בזמן אמת: כמה הודעות נשלפו, אילו זיכרונות, latency. זה כלי התצפית העיקרי, החליף את הלוגים.

> ⚠️ הקוד לא עושה hot-reload. כל שינוי קוד דורש restart של `maya web` מקומית, או `make deploy.redeploy` ל-Railway.
